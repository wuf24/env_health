# Lancet 未来情景预测框架说明

这份说明对应 `6 未来情景分析/scripts/run_future_scenario_projection.py`，目标是把 `5 反事实推演` 里已经成熟的固定效应建模思路，改造成更接近参考 Lancet 文章 Figure 5 的“前向情景预测”框架。

## 一句话概括

不是把未来情景分析简单理解成“反事实的反方向”，而是：

1. 先用历史面板估计协变量与 AMR 的关联系数；
2. 再用 ETS 给 outcome 本身做一条“基线趋势延续”；
3. 最后把未来情景下协变量相对基线路径的变化，按历史系数折算成情景调整项，加回基线趋势。

所以它和反事实推演很像，但不完全相同。

## 与 `5 反事实推演` 的关系

`5 反事实推演` 的核心是：

```text
历史观测值 -> 选主模型 -> 用同一套 FE 系数做基准替换 -> 比较 actual 与 counterfactual
```

这里改成：

```text
历史观测值 -> 选主模型 -> 用同一套 FE 系数做未来情景调整 -> 比较 baseline 与 future scenario
```

也就是说，保留了：

- 既有模型筛选结果
- 固定效应主模型与稳健性模型
- 历史标准化尺度
- 省级面板结构

替换掉的是：

- “把某些变量恢复到基准年”的反事实设定
- 改为“让某些变量沿未来情景路径前进”的前向预测设定

## 严格参考 Lancet 文章时，对应的关键步骤

参考文章公开文本中可以确认的要点是：

1. 用历史多国面板做固定效应回归。
2. 基线情景不是直接把回归方程一路往前算，而是用 ETS 延续当前趋势。
3. 不同 SSP、单因素控制、多因素控制情景，再在这个基线之上比较差异。
4. 最后若要做疾病负担，还要把预测的耐药流行率继续换算成死亡或 YLL。

在本项目里，对应成：

### 第一步：历史关联模型

```text
Y_it = α_i + λ_t + Σ_k β_k Z_itk + ε_it
```

- `Y_it`：历史 AMR 指标
- `α_i`：省固定效应
- `λ_t`：年固定效应
- `Z_itk`：按历史样本均值和标准差标准化后的协变量
- `β_k`：历史面板里识别出来的关联系数

### 第二步：基线趋势

```text
Y^base_it = ETS(Y_i,1...T)
```

也就是先对每个省的历史 AMR 自身做 ETS 外推，得到 2024–2050 的“趋势延续”基线。

### 第三步：未来情景调整

```text
Δ^scenario_it = Σ_k β_k × ( Z^scenario_itk - Z^base_itk )
```

### 第四步：未来情景预测值

```text
Y^scenario_it = Y^base_it + Δ^scenario_it
```

这一步就是 Lancet 风格下最核心的“基线 + 情景偏移”。

## 当前仓库里先做成了什么

当前已经能立即运行的是：

- 历史 outcome：`AMR_AGG` 或单一 AMR 指标
- 历史协变量：沿用 `climate_social_eco.csv`
- 未来外部情景路径：CCKP 中国省级逐年 `rx1day`
- 基线方法：省级 ETS
- 情景组：`ssp119 / ssp126 / ssp245 / ssp370 / ssp585`
- 不确定性路径：`median / p10 / p90`

## 为什么当前先只把 `rx1day` 接进来

因为你现在已经实际落地并核验过的未来省级逐年变量，只有：

- `R1xday`

而 Lancet 文章里的未来情景还涉及：

- 抗菌药物使用
- 健康支出
- 饮水服务
- SSP 社会经济路径

这些在你当前仓库里还没有同口径省级未来路径，所以这次先把框架搭好：

- 让 `R1xday` 先真正跑起来
- 其他协变量先沿基线延续
- 后面再逐个往里补未来输入表

## 为什么做了外部气候路径与历史观测的对齐

CCKP 的逐年 `rx1day` 来自气候模型集合，并不等于你历史面板里的观测值。

因此脚本默认做了一个稳妥处理：

- 用历史重叠年份计算每个省在每个 SSP/统计量下的平均偏差
- 把未来 `rx1day` 路径先做 bias correction
- 再拿去和历史口径下的基线路径比较

这样比“直接生硬拼接”更稳健。

## 如果后面要继续贴近 Lancet Figure 5

下一步最值得补的不是再改公式，而是补输入：

1. 省级未来抗菌药物使用路径
2. 省级未来水/卫生条件路径
3. 省级未来经济与卫生投入路径
4. 若要走到死亡负担，还要补省级感染死亡数和 RR

一旦这些输入补齐，这套框架就能很自然地扩展到：

- 单因素控制情景
- 多因素联合控制情景
- 2050 情景比较图
- 死亡负担与经济负担模块
